About Me

My journey, goals, and reflections

My Bio

I am a passionate frontend developer with over 3+ years of experience building web applications. My journey started with basic HTML and CSS, and I've since expanded my skills to include modern Javascript frameworks, responsive design, and accessibility best practices.

I believe in creating web experiences that are not only visually appealing but also inclusive and performant. I'm constantly learning and staying up-to-date with the latest web technologies and best practices.

Goals in This Program

During this internship I want to work toward three concrete outcomes I can point to after the program:

  • Ship a small, accessible UI component library (3–5 components) with documented usage and keyboard/ARIA support.
  • Improve page load performance by learning and applying code-splitting and image optimization techniques — aim to reduce a demo page's First Contentful Paint by 30%.
  • Write at least five focused unit/integration tests for a feature I build, and integrate them into a simple CI check.

These goals are intentionally measurable: they help me learn tooling, accessibility, and quality practices that make real projects more maintainable.

Areas of Low Confidence

There are a few practical skills I want to build with focused practice and mentoring:

  • CSS systems & architecture: I know layout and components, but I want to learn patterns for scale — design tokens, utility layers, and component APIs that keep styles predictable in large apps.
  • Performance diagnostics: I'm comfortable improving images and bundles, but I need more experience using tooling (Lighthouse/DevTools/Perf profiles) to find the real hotspots and prioritize fixes.
  • Test-driven development: I can write tests, but struggle to design tests that focus on behavior rather than implementation — I want to practice writing stable, maintainable tests.
  • Deployment pipelines: I've not yet set up CI/CD for a project from scratch; learning a minimal pipeline (GitHub Actions) for linting, tests, and static hosting is a priority.

Note to Future Self

When you read this in six months, check whether you shipped the component library and the tests you planned. If not, ask: what roadblock stopped you and what can you do this week to unblock it?

Celebrate small wins — a PR merged, a failing test fixed, or a page that loads noticeably faster. Those are the signals that you're improving. Keep a short weekly log (3 bullets) describing what you learned and one thing you'll do differently next week.

Finally, be kind to yourself. Growth is a series of tiny improvements. Focus on clarity, consistency, and sustainable momentum rather than perfection.

Extra Thoughts

Frontend development is more than just writing code - it's about creating experiences. Every decision we make, from the colors we choose to the way we structure our components, impacts how users interact with our applications.

I'm excited to be part of a community that values not just technical excellence but also collaboration, mentorship, and continuous improvement. The web is an incredible platform that continues to evolve, and i'm grateful to be part of shaping its future.
